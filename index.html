What it does

Everything you need.
Nothing you don't.

⚡
Profile-Based Config

Create separate header profiles for each environment — staging, local, production. Switch between them instantly from the popup.

🔀
Request & Response

Modify both outgoing request headers and incoming response headers. Inject auth tokens, fix CORS issues, override content types.

🎯
URL Filtering

Scope rules to specific domains using pattern matching. Apply headers only to *.staging.myapp.com while leaving everything else untouched.

🔧
Three Operations

Set replaces or creates a header. Append adds to existing values. Remove strips headers entirely — useful for stripping CSP or auth headers during testing.

📦
Built-in Presets

One-click templates for the most common setups: Auth Bearer, CORS Allow-All, JSON Accept, No-Cache, and custom User-Agent.

💾
Persistent Storage

All profiles and headers are saved to Chrome's local storage. Your config survives browser restarts, updates, and system reboots.

How it works

Common questions.

How do I add a new header?

Open the popup, select a profile (or create one), and click + Add Header. Fill in the header name, value, choose whether it applies to requests or responses, and set the operation. Hit Apply (or Ctrl+Enter) to activate.

Can I limit headers to one site?

Yes. Each profile has a URL Filter field. Enter a pattern like *.staging.myapp.com or *://api.example.com/*. Use * to apply to all URLs.

What's the difference between set, append, and remove?

Set creates or overwrites a header with the given value. Append adds the value to any existing header value. Remove strips the header entirely — no value needed.

Do I need to reload tabs after applying?

Yes. declarativeNetRequest rules apply to new requests. After clicking Apply, reload the page or tab you want the rules to take effect on.

Can I have multiple profiles active at once?

Currently one profile is active at a time. Switch between profiles in the tab bar at the top of the popup. You can toggle individual headers on/off within a profile without switching.

Is my data private?

Completely. HeaderForge stores all config in Chrome's local storage — it never leaves your browser. No telemetry, no servers, no analytics.
